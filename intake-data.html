Prototype · Agentic AI

Intake & Triage Agent

Structured forms make users categorize their own problem before they've gotten any help. This prototype replaces the form with freeform conversation — the agent detects intent, asks one targeted follow-up, and routes the case autonomously.

Problem

▼

Standard intake forms put the categorization burden on the user — asking them to select a department, choose a topic type, and articulate their own problem before getting any help. Users who are frustrated or in a hurry pick the wrong category, skip required fields, or abandon the form entirely. The downstream effect is misrouted tickets, higher handle times as agents re-establish what the actual issue is, and duplicated cases from customers who resubmit because they never heard back.

The core assumption embedded in a structured form is that the user knows what kind of problem they have. Often they don't — they just know something is wrong.

Constraints

▼
  • The flow had to handle freeform input across six distinct intent types without requiring the user to choose a category
  • Intent detection had to work in a single turn for common inputs — too many clarifying turns would recreate the friction of a structured form
  • Follow-up questions had to be targeted to the detected intent: billing cases asked for an amount or date, technical cases for a device — generic follow-ups added noise without improving the case record
  • Urgency signals (emergency, ASAP, critical, right now) needed to override standard priority assignment regardless of detected intent
  • The case record had to be immediately actionable downstream — intent, priority, routed team, and contact info all captured in a single intake pass

Product Ownership

▼

Conversational Flow Design

Designed the three-turn intake structure — describe the issue, answer one targeted follow-up, provide contact info. Follow-up question and input placeholder are dynamically set by detected intent, so billing cases are asked for an amount and technical cases are asked for a device.

Intent Detection & Classification

Defined keyword sets for six intent types (complaint, billing, technical, shipping, feature request, compliment) and the urgency detection layer that operates independently of intent. Specified fallback behavior for low-confidence and general inputs.

Agent Action Sequence

Designed the visible processing steps — reading, detecting, assessing, routing, creating case — with animated status indicators so the user can observe the agent's decision-making rather than seeing a result appear without context.

Case Record Design

Defined the structured output: detected intent, priority level, urgency flag, routed team, case number, timestamp, and full user input retained for reference. Output is written to sessionStorage to power the separate data view.

Outcome

▼

The freeform intake approach captured intent accurately without requiring the user to self-categorize. Targeted follow-up questions — dynamically set by detected intent — extracted the specific detail most useful for each case type in a single additional turn. The agent reduced intake to three user turns, producing a case record ready for immediate team handoff with no manual re-reading or recategorization required.
