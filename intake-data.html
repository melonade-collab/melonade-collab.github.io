Intake & Triage Agent

Structured forms make users categorize their own problem before they've gotten any help. This prototype replaces the form with freeform conversation — the agent detects intent, asks one targeted follow-up, and routes the case autonomously.

Problem

▼

Standard intake forms put the categorization burden on the user — asking them to select a department, choose a topic type, and articulate their own problem before getting any help. Users who are frustrated or in a hurry pick the wrong category, skip required fields, or abandon the form entirely. The downstream effect is misrouted tickets, higher handle times as agents re-establish what the actual issue is, and duplicated cases from customers who resubmit because they never heard back.

The core assumption embedded in a structured form is that the user knows what kind of problem they have. Often they don't — they just know something is wrong.

Tech

▼
  • Keyword-based intent classifier covering six intent types: complaint, billing, technical, shipping, feature request, compliment
  • Urgency layer (keywords: emergency, ASAP, critical, right now) operates independently — overrides priority regardless of detected intent
  • Per-intent follow-up questions and input placeholders set dynamically at runtime so each intent gets a targeted clarifying ask
  • Animated five-step processing sequence (reading → detecting → assessing → routing → creating case) renders the agent's decision-making visible rather than instant
  • sessionStorage used to write the case record progressively and power the separate data view without a backend

Features

▼
  • Freeform first message — user describes the issue in natural language, no category selection required
  • Intent detection in a single turn with fallback for low-confidence or general inputs
  • One targeted follow-up per intent: billing cases are asked for an amount or date, technical cases for a device, shipping for an order number
  • Urgency override: emergency signals raise priority to urgent regardless of intent type
  • Case record output: intent, priority, urgency flag, routed team, case number, timestamp, and full user input — immediately actionable for downstream handoff

Why This

▼

Structured forms assume users know what kind of problem they have — often they don't, they just know something is wrong. Asking people to self-categorize before getting help creates the first failure point: the wrong category leads to misrouting, longer handle times, and duplicate cases from customers who never get a response. Freeform intake moves classification to the agent, where it belongs.
