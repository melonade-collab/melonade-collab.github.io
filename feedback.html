Feedback Routing Agent

Most feedback forms dump submissions into a queue where someone manually reads, tags, prioritizes, and assigns each one. This prototype explores what that intake layer looks like when an agent does it instead.

The Problem

▼

Support teams spend real time on triage — not resolution. Every submission has to be read, categorized, assessed for urgency, and routed before any actual help begins. At volume, this gets inconsistent.

Tech

▼
  • Client-side NLP for sentiment detection and extractive summarization
  • Address geocoding via a live lookup API (Nominatim / OpenStreetMap)
  • Multi-step agent pipeline with animated state transitions
  • Structured JSON output for downstream system handoff

Features

▼
  • Live sentiment detection — reads tone as you type (positive / negative / neutral), with negation handling so "not great" doesn't register as positive
  • Extractive summarization — distills long messages to the key sentence so you can confirm what the agent will act on before submitting
  • Open case detection — looks up the email or phone and surfaces any existing open cases, with the option to link the submission rather than create a duplicate record
  • Address autocomplete — enriches submissions with location data via a live geocoding API for regional routing or geo-clustering
  • NPS-style rating — 0–10 scale with contextual labels per score that adjust based on the selected value
  • Agent routing pipeline — animated multi-step processing: reads feedback → detects category and intent → assesses priority → routes to the right team
  • Case creation — generates a structured case record with priority level, category, assigned queue, and SLA
  • Data transparency — shows the full JSON payload so you can see exactly what gets sent downstream

Why This

▼

Every feedback form creates a triage problem — someone has to read each submission, work out what it's about, judge the urgency, and route it before anything useful happens. This prototype puts that cognitive work on the agent, and runs it client-side before anything reaches a queue. The browser is doing the reading, the categorizing, and the routing decision.
