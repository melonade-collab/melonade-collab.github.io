Conversational Feedback Agent

Static feedback forms collect every field upfront before any interaction. This prototype replaces the form with a conversation — collecting contact info, rating, and message through natural dialogue with skip, retry, and edit at every step.

Problem

▼

Static feedback forms collect every field upfront — name, contact details, rating, message — before any interaction. Users who are in a hurry, unsure what to write, or reluctant to share personal details abandon halfway through or fill in the minimum. What gets submitted arrives as unstructured text needing manual review and routing before anything can be done with it.

Tech

▼
  • Conversational state machine with nine steps (name, city, province, email, phone, postal, rating, message, review) and skip, retry, and edit paths at each step
  • City-to-province lookup table covering major Canadian and US cities — used for a soft-confirm flow that auto-suggests province without a dropdown
  • Client-side sentiment analysis run on the message field
  • sessionStorage used for real-time data persistence so the data view page reads live session state without requiring a submission event

Features

▼
  • Nine-step conversational intake covering all fields a traditional form would collect
  • At each step: skip detection, retry prompts, quit/cancel handling, privacy question responses, and confusion recovery
  • Review step at the end where users can change any field by naming it — re-asks only that specific step
  • Address intelligence: recognized cities auto-suggest province with a soft confirm; city/province mismatches trigger a natural correction flow
  • Rating accepts numeric input, approximate language ("maybe a 7"), and out-of-range values with natural correction

Why This

▼

Conversation matches how people naturally report issues — they describe, qualify, and correct as they go. Removing the upfront form eliminates the cognitive load of deciding what to include before any dialogue has started. Progressive collection with skip options lowers the barrier to starting, while the edit flow means users aren't locked into their first answers.
